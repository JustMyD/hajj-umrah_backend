# Init Database Scripts

Эта директория предназначена для скриптов инициализации базы данных PostgreSQL.

## Как это работает

PostgreSQL Docker образ автоматически выполняет все `.sql` и `.sh` файлы из директории `/docker-entrypoint-initdb.d/` при первом запуске контейнера (когда база данных пустая).

Скрипты выполняются в алфавитном порядке, поэтому используйте префиксы с номерами для контроля порядка выполнения (например, `01-create-user.sh`, `02-create-schema.sql`).

## Текущая конфигурация

В данный момент используется пользователь `postgres` для подключения к базе данных. 

Если в будущем понадобится создать отдельного пользователя для приложения, добавьте скрипт в эту директорию и смонтируйте её в `docker-compose.yml`:

```yaml
volumes:
  - ./init-db:/docker-entrypoint-initdb.d
```

### Важно

⚠️ **Скрипты выполняются только при первом запуске контейнера!** Если база данных уже существует, скрипты не будут выполнены.
